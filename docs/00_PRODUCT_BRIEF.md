# 00 — Product Brief

## Product name

Codex PM

## One-line positioning

An open-source Codex-native project manager plugin that turns `docs/` Markdown project documents into verified development progress.

## Core concept

Codex PM is not a replacement for Codex. It is a **cerebellum-style project manager layer** that controls task selection, risk, cost, memory, verification, and adaptive strategy.

```text
Codex = LLM brain + engineer
Codex PM = PM cerebellum + memory + risk + verification + strategy
```

## Why this exists

Codex is strong at single tasks, but real projects require:

- task sequencing,
- dependency awareness,
- scope control,
- cost control,
- risk gating,
- verification,
- failure recovery,
- long-term memory,
- feedback-based strategy improvement.

Codex PM turns these into a local, open-source project manager plugin.

## Slogan

```text
Docs in. Verified progress out.
```

## Differentiation

- Not a general AI task manager.
- Not a PRD-to-task generator only.
- Not another LLM agent.
- It is a Codex-native delivery control layer.

## Research angle

Codex PM also acts as an experiment platform for testing whether a small-brain agent architecture can adapt and evolve in real software development tasks.

The experiment question:

> Can a non-LLM cerebellum agent improve its project delivery performance through memory, feedback, reward, risk, cost, and strategy adaptation while using Codex as its brain?
